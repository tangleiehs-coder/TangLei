# Get笔记自动标题方案总结

## 当前状况

### 1. 纯本地提取版（已完成）
**文件**: `test_title_final.py`

**效果**: 能提取一些关键词，但**不够智能**，经常出现：
- 提取了旁支细节而非核心观点
- 标题长度控制不稳定
- 对口语化记录（如孩子日常）提取不准

**建议**: 作为备选方案，不需要API key

---

### 2. Kimi API 版（推荐）
**文件**: `getbiji_auto_title.py`（需要完善）

**优势**: 
- 调用Kimi模型，能真正理解内容
- 生成"是什么事+什么观点"的精准标题
- 支持自定义提示词，符合你的R-B-A-T-O风格

**使用方式**:
```python
# 设置API Key
export KIMI_API_KEY="sk-..."

# 运行
python3 getbiji_auto_title.py
```

---

## 建议方案

### 方案A：使用 Get 笔记自带的 AI 功能
Get笔记本身有AI助手（右侧边栏），可以直接问它：
> "给这条笔记起个标题，一眼能看出是什么事、什么观点"

**优点**: 无需脚本，即开即用
**缺点**: 不能批量处理，需要手动操作

---

### 方案B：用 Kimi Claw/OpenClaw 手动整理
既然你已经在用 Kimi Claw，可以直接让它帮你：
1. 把无标题笔记的内容贴给它
2. 让它按你的风格生成标题
3. 你复制粘贴回去

---

### 方案C：完善自动化脚本（需要投入）
如果要全自动，需要：
1. ✅ 安装 Playwright（浏览器自动化）
2. ✅ 配置 Kimi API Key
3. ✅ 调试标题生成提示词
4. ✅ 设置定时任务

**预计耗时**: 1-2小时调试

---

## 瓜瓜建议

先别折腾脚本了。**Get笔记自带的AI其实够用**。

如果真的想自动化，建议：
1. 先用方案B（手动让Kimi生成标题）跑一周
2. 看看生成的标题是否符合你的预期
3. 满意了再写成自动化脚本

你现在最缺的不是自动化工具，而是**确定什么样的标题对你最有用**。

要我帮你写一个"手动版"的标题生成提示词吗？你直接贴给 Kimi 用。
