# 课程视觉工程统一协议（U-CVEP）

## 一、核心金线（不可违背的底层法则）

**唯一定义**：所有视觉元素必须是认知翻译器，而非装饰物。

### 三强制律

1. **语义保真律**：关闭音频后，视觉独立传达的信息与音频原意偏差不得超过5%
2. **语境锁定律**：视觉生成必须基于功能角色和语义粒度双重标记，禁止基于孤立关键词生成
3. **相位滞后律**：概念类视觉必须滞后音频0.3-0.5秒出现，禁止视觉剧透

---

## 二、预处理阶段：文本解构协议

### 2.1 双重标记系统（必须同时完成）

对逐字稿逐句标注：

**A. 功能色标（决定视觉策略）**
- 🔴 **概念体（Concept）**：首次定义术语（需隐喻化视觉锚点）
- 🟡 **叙事流（Narrative）**：案例/故事（需场景连续性）
- 🟢 **对比场（Conflict）**：对错/前后对比（需空间并置或切换）
- 🔵 **操作链（Action）**：步骤/方法（需流程渐进展示）
- ⚪ **元认知（Meta）**：过渡/总结（需极简或渐隐）

**B. 粒度层级（决定抽象程度）**
- **L1宏观**：理念/价值观 → 必须配符号隐喻（容器/种子/网络等）
- **L2中观**：模型/框架 → 必须配结构图示（流程图/矩阵/层级图）
- **L3微观**：具体动作/对话 → 必须配情境还原（人物/空间/动作）

### 2.2 时间窗规划（Temporal Windowing）

将课程切分为语义窗：
- **概念窗**：从定义开始到举例结束（通常30-60秒）
- **叙事窗**：从场景切入到结论提炼（保持完整性，禁止切割）
- **对比窗**：A/B双轨并行展示（强制分屏或快速切换）

**窗内控制**：
- 每 **10±2秒** 必须有一次视觉刷新（M1/M2级动态）
- L3高负荷内容可延长至15秒，但必须在10秒处插入焦点呼吸（关键元素脉动高亮）

## 三、生成阶段：视觉构建与对齐协议

### 3.1 生成前置校验（Alignment Checklist）

生成每个视觉前，必须填写：

```
【视觉生成工单】
目标片段：[原文]
功能色标：[🔴/🟡/🟢/🔵/⚪]
粒度层级：[L1/L2/L3]
语义窗口：[00:00-00:45]

对齐四问：
1. 核心语义：（用抽象语言概括，禁止复制原文关键词）
2. 功能角色：（在课程结构中承担什么作用？承接什么？开启什么？）
3. 粒度匹配：（当前是理念层/方法层/操作层？）
4. 相位策略：[负相位-0.5s/正相位0s/预相位+2s]

视觉语法检查：
- 若为🔴概念体：是否使用了标准隐喻？（容器/种子/聚光灯等）
- 若为🟡叙事流：是否保持了空间连续性？
- 若为🟢对比场：是否使用了空间并置？
- 若为🔵操作链：是否采用了渐进揭示？
```

### 3.2 视觉语法矩阵

根据功能色标和粒度层级的交叉，调用对应视觉策略：

| | L1宏观（理念） | L2中观（模型） | L3微观（场景） |
|:---|:---|:---|:---|
| **🔴概念体** | 抽象符号隐喻（容器倾倒/种子破土） | 概念关系图（网络节点/同心圆） | 概念具象化（巨大化符号介入现实场景） |
| **🟡叙事流** | 场景氛围渲染（色调/光影暗示情绪） | 角色关系拓扑（人物位置/连线） | 连续镜头语言（焦点推拉跟随对话） |
| **🟢对比场** | 左右分屏（冷vs暖，封闭vs开放） | 前后状态对比（同一空间的时间切片） | 动作对比（错误姿势vs正确姿势并置） |
| **🔵操作链** | 流程总览图（仅展示骨架） | 步骤拆解（逐步高亮当前步骤） | 第一人称视角（POV）展示具体操作 |
| **⚪元认知** | 极简留白（纯色背景+呼吸文字） | 思维导图聚合（前述内容缩略聚合） | 黑场/淡出（给认知休息） |

### 3.3 动态节奏控制（Rhythm Control）

**三级动态标准**：
- **M0微动（0-2秒）**：元素内部动画（呼吸灯/文字高亮），用于维持注意力
- **M1重组（5-10秒）**：画面元素位置调整、视角切换，用于推进逻辑
- **M2转场（15-30秒）**：完整场景变换，仅用于语义窗切换或章节转换

**相位规则**：
- **负相位（-0.5s）**：概念体（🔴）和对比场（🟢）使用，视觉滞后音频
- **正相位（0s）**：叙事流（🟡）的动作描述使用，音画同步
- **预相位（+2s）**：仅用于M2转场，视觉先出现建立新场景

## 四、品控阶段：验证与修正协议

### 4.1 三重强制性检测

**检测A：静默语义测试（Silent Semantic Test）**
- 关闭音频，仅看视觉序列
- 随机抽取5个时间点，描述"这段在讲什么"
- **通过标准**：描述与音频原意语义一致
- **致命缺陷**：理解偏差（如将"温和引导"理解为"严厉批评"）

**检测B：粒度对齐扫描（Granularity Scan）**
- 检查每个视觉单元的粒度标签与实际视觉风格是否匹配
- **红线**：L1配了写实照片/L3配了抽象符号/同一窗口内粒度跳跃无过渡

**检测C：相位偏移检测（Phase Detection）**
- 视觉动态峰值必须滞后音频语义峰值 0.3-0.8秒
- **缺陷**：视觉提前于音频出现（剧透）或滞后超过1.5秒（脱节）

### 4.2 缺陷分级与处置

| 等级 | 判定标准 | 处置 |
|:---|:---|:---|
| **致命** | 违反核心金线（语义偏差/视觉剧透/认知干扰） | 立即停产，追溯返工 |
| **严重** | 粒度错配/功能色标与视觉策略不符/节奏断裂 | 48小时内修正 |
| **轻微** | 微动幅度不均/色调偏差 | 迭代优化 |

## 五、系统提示词模板

```
你是一位课程视觉工程师，严格执行U-CVEP协议。按以下流程操作：

步骤1【解构】：
读取逐字稿，逐句标记：
- 功能色标：[🔴概念体/🟡叙事流/🟢对比场/🔵操作链/⚪元认知]
- 粒度层级：[L1宏观/L2中观/L3微观]
- 划分语义窗（Concept Window/Narrative Window/Contrast Window）

步骤2【规划】：
对每个语义窗：
- 规划窗口起止时间码
- 确定窗内动态节奏（每10±2秒一次M1/M2刷新）
- 标记相位策略（概念用负相位-0.5s，叙事动作用正相位0s，转场用预相位+2s）

步骤3【生成】：
对每个时间点生成视觉时，先填写【视觉生成工单】，然后：
- 查《视觉语法矩阵》（功能色标×粒度层级=视觉策略）
- 调用《视觉锚点词典》获取标准隐喻
- 生成包含相位信息的视觉描述

步骤4【校验】：
自检三项红线：
□ 静默测试：关闭音频看视觉，理解是否与音频一致？
□ 粒度测试：L1不写实，L3不抽象，无粒度跳跃
□ 相位测试：无视觉峰值超前音频（防剧透）

输出格式：
时间码 | 原文 | 功能-粒度 | 相位 | 视觉策略简述 | 生图Prompt（含动态描述）

禁止事项：
- 禁止基于孤立关键词生成视觉
- 禁止在🔵操作链中一次性展示全部步骤
- 禁止在🟡叙事流中频繁切镜
- 禁止在🔴概念体中使用负向词汇字面化
```
